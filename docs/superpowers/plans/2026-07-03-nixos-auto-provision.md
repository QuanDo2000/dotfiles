# Near-zero-touch NixOS provisioning — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make a fresh NixOS box provision in one command by auto-detecting per-machine values (username/hostname/timezone/stateVersion), confirming them once, and channel-guarding packages so a rebuild can't eval-fail.

**Architecture:** `configuration.nix` stops hardcoding per-machine values — it reads them from `/etc/nixos/machine.nix` (imported by absolute path, symlink-safe) and imports `/etc/nixos/hardware-configuration.nix` in place. A shared `_nixos_ensure_linked` helper detects values, confirms them when a TTY is present, writes `machine.nix` (first run only), and symlinks the repo config into `/etc/nixos`; `install_nixos`/`update_nixos` call it before rebuilding. Channel-sensitive packages (ghostty, opencode) are guarded with `lib.optional (pkgs ? name)`.

**Tech Stack:** Bash (sourced scripts), NixOS/nixpkgs, the repo's bash test harness under `tests/bash/`.

## Global Constraints

- Bash 3.2 compatible (no `mapfile`/`readarray`). Logging via `info`/`success`/`fail`/`user` from `utils.sh`.
- Every system-mutating command is gated behind `[[ "$DRY" == "false" ]]`; `DRY=true` logs but changes nothing (no `sudo`, no `tee`, no `ln`, no `nixos-rebuild`).
- `machine.nix` lives at `/etc/nixos/machine.nix`, imported by ABSOLUTE path. The path is overridable via `NIXOS_MACHINE_FILE` (default `/etc/nixos/machine.nix`) for tests.
- Interactive confirm runs ONLY when `[[ -t 0 && "$DRY" == "false" ]]`. Non-TTY (tests/CI/piped) uses detected values without prompting.
- `machine.nix` is generated only when missing; `FORCE` (`-f`) regenerates it. Existing file → left untouched.
- Detection fallbacks: username→`nixos`, hostName→`nixos`, timeZone→`UTC`, stateVersion→`24.11`. Detection never hard-fails.
- `configuration.nix` must parse (`nix-instantiate --parse`) even though `/etc/nixos/machine.nix` and the hardware config don't exist in the repo (`--parse` doesn't resolve imports).
- Commits are GPG-signed; if `git commit` hits a GPG/TTY error, stage and report DONE_WITH_CONCERNS for the controller to commit.

```bash
# Body of /etc/nixos/machine.nix — the per-machine values configuration.nix
# reads. Pure (no I/O) so it is unit-testable.
# Usage: _nixos_machine_file_content <username> <hostName> <timeZone> <stateVersion>
_nixos_machine_file_content() {
  cat <<EOF
{
  username = "$1";
  hostName = "$2";
  timeZone = "$3";
  stateVersion = "$4";
}
EOF
}

# Raw \`nixos-version\` output (empty on failure). Wrapped in a function so tests
# can override it — the command name contains a hyphen, so it can't be mocked
# as cleanly as \`hostname\`.
_nixos_version_string() { nixos-version 2>/dev/null || true; }

# Detect per-machine values from the running system. Echoes four lines:
# username, hostName, timeZone, stateVersion. Every field has a fallback so
# detection never hard-fails.
_detect_nixos_machine_values() {
  local u h t v
  u="${SUDO_USER:-$(whoami 2>/dev/null || true)}"
  [[ -n "$u" ]] || u="nixos"
  h="$(hostname 2>/dev/null || true)"
  [[ -n "$h" ]] || h="nixos"
  t="$(timedatectl show -p Timezone --value 2>/dev/null || true)"
  [[ -n "$t" ]] || t="UTC"
  v="$(_nixos_version_string | cut -d' ' -f1 | cut -d. -f1,2)"
  [[ -n "$v" ]] || v="24.11"
  printf '%s\n%s\n%s\n%s\n' "$u" "$h" "$t" "$v"
}
```

- [ ] **Step 3: Implement `_prompt_default` + `_nixos_ensure_linked`, rewrite the two functions**

```bash
# Prompt for a value showing a detected default; empty input keeps the default.
# The prompt goes to stderr (via read -p) so stdout stays clean for capture.
# Only ever called under an interactive TTY. Usage: _prompt_default <label> <default>
_prompt_default() {
  local label="$1" default="$2" answer
  read -rp "  [ ?? ] $label [$default]: " answer
  echo "${answer:-$default}"
}

# Ensure /etc/nixos/machine.nix exists and the repo configuration.nix is linked
# into /etc/nixos. On first run (machine.nix missing, or FORCE), detect the
# per-machine values, confirm them interactively when a TTY is attached, and
# write the file. hardware-configuration.nix in /etc/nixos is left untouched.
_nixos_ensure_linked() {
  local mf="${NIXOS_MACHINE_FILE:-/etc/nixos/machine.nix}"
  local cfg="$DOTFILES_DIR/config/nixos/configuration.nix"

  if [[ ! -f "$mf" || "$FORCE" == "true" ]]; then
    info "Detecting machine settings..."
    local u h t v
    { read -r u; read -r h; read -r t; read -r v; } < <(_detect_nixos_machine_values)
    if [[ -t 0 && "$DRY" == "false" ]]; then
      u="$(_prompt_default "Username" "$u")"
      h="$(_prompt_default "Hostname" "$h")"
      t="$(_prompt_default "Timezone" "$t")"
      v="$(_prompt_default "NixOS stateVersion" "$v")"
    fi
    if [[ "$DRY" == "false" ]]; then
      _nixos_machine_file_content "$u" "$h" "$t" "$v" | sudo tee "$mf" >/dev/null \
        || fail "Failed to write $mf"
      success "Wrote machine settings to $mf"
    else
      info "Would write $mf (username=$u hostName=$h timeZone=$t stateVersion=$v)"
    fi
  else
    info "Using existing $mf"
  fi

  if [[ "$DRY" == "false" ]]; then
    [[ -f "$cfg" ]] || fail "NixOS config not found: $cfg"
    sudo ln -sfn "$cfg" /etc/nixos/configuration.nix \
      || fail "Failed to link configuration.nix into /etc/nixos"
  fi
}
```

Now replace the existing `install_nixos` and `update_nixos` function bodies (keep/adjust their leading comments) with:

```bash
# Reprovision NixOS: ensure machine.nix + the /etc/nixos/configuration.nix
# symlink (see _nixos_ensure_linked), then nixos-rebuild switch. On NixOS all
# imperative setup_* installers are skipped — packages come from the rebuild.
# Usage: install_nixos
function install_nixos {
  info "Installing packages for NixOS..."
  _nixos_ensure_linked
  if [[ "$DRY" == "false" ]]; then
    sudo nixos-rebuild switch || fail "nixos-rebuild switch failed"
  fi
  success "Finished install for NixOS"
}

# Update NixOS: ensure the config is linked, then rebuild with channel upgrade.
# Usage: update_nixos
function update_nixos {
  info "Updating packages for NixOS..."
  _nixos_ensure_linked
  if [[ "$DRY" == "false" ]]; then
    sudo nixos-rebuild switch --upgrade || fail "nixos-rebuild switch --upgrade failed"
  fi
  success "Finished update for NixOS"
}
```
