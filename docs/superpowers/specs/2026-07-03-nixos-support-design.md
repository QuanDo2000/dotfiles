# Preliminary NixOS support

**Date:** 2026-07-03
**Status:** Approved, ready for implementation plan

## Goal

Add preliminary NixOS support to the dotfiles. On NixOS the machine is owned
declaratively by a tracked `configuration.nix`; `dotfile packages` reprovisions
the system via `nixos-rebuild switch`. This is a *preliminary* pass: NixOS gets
recognized and wired end-to-end, and a full-desktop config template lands, but
the `.nix` file is a starting point that needs per-machine tuning.

## Decisions

- **Own the whole system** via `configuration.nix` (not home-manager).
- **Full desktop** scope: packages + core system + Hyprland/greetd/fcitx5/fonts.
- **App configs stay as symlinked dotfiles.** NixOS installs the packages and
  enables the services; the actual `.conf` files under `config/unix/` keep being
  symlinked into `~/.config` by `dotfile symlinks`, unchanged. No config is
  rewritten into Nix. This keeps the configs shared with the Arch/Mac setups.
- **`dotfile packages` reprovisions:** symlink the tracked config into
  `/etc/nixos/` and run `sudo nixos-rebuild switch`.
- **Classic non-flake config** for this preliminary pass. `nixos-rebuild switch`
  reads `/etc/nixos/configuration.nix` directly with no boilerplate. Flakes are a
  later upgrade; `nix.settings.experimental-features` is set now so the switch is
  cheap later.

## Design

### 1. Platform detection — `scripts/platform.sh`

Add a `nixos` branch to `detect_platform`, checked **before** the debian/arch
branches. NixOS's `/etc/os-release` has `ID=nixos` (and `ID_LIKE` is unset), so
it won't collide with the existing checks, but ordering it first keeps the intent
explicit. Returns the string `nixos`. All downstream `case` statements switch on
it.

### 2. Tracked config — `config/nixos/configuration.nix`

One file, a full-desktop starting template:

- **System core:** `system.stateVersion`, `nix.settings.experimental-features =
  [ "nix-command" "flakes" ]`, `time.timeZone`, `i18n.defaultLocale`,
  `networking.hostName`, and `imports = [ ./hardware-configuration.nix ]`.
- **User:** the user account with `shell = pkgs.zsh` and `programs.zsh.enable =
  true`.
- **Desktop:** `programs.hyprland.enable`, greetd as the login manager, FiraCode
  Nerd font via `fonts.packages`, `i18n.inputMethod` = fcitx5, plus waybar /
  ghostty / wl-clipboard in packages.
- **`environment.systemPackages`:** the `ARCH_PACKAGES` list ported to nixpkgs
  attribute names — git, zsh, tmux, neovim, fzf, fd, ripgrep, lazygit, jujutsu,
  starship, zoxide, gnupg, openssh, etc. This replaces every imperative `setup_*`
  installer on NixOS.

`hardware-configuration.nix` is **git-ignored** — it is machine-generated by
`nixos-generate-config` and is per-machine. The tracked file only imports it.

Per-machine knobs (hostname, username, timezone, hardware import) are marked with
comments; they are filled in on a real machine.

### 3. `dotfile packages` on NixOS — `scripts/packages.sh`

New `install_nixos` and `update_nixos`, wired into the `install_packages` /
`update_packages` `case` statements:

```bash
sudo ln -sfn "$DOTFILES_DIR/config/nixos/configuration.nix" \
  /etc/nixos/configuration.nix
sudo nixos-rebuild switch
```

Symlink (not copy) so editing the tracked file and re-running just works, matching
the repo's symlink philosophy. `/etc/nixos/hardware-configuration.nix` is left
untouched. **All the imperative installers** (`setup_neovim`, `setup_lazygit`,
`setup_jj`, `setup_starship`, `setup_opencode`, the curl-script tools) are
**skipped** on NixOS — they download dynamically-linked binaries that don't run
under NixOS's non-FHS layout; packages come from the rebuild instead.

Both functions respect `DRY` (log, don't run) like the existing installers.

### 4. Unchanged behavior

- `dotfile symlinks`, `dotfile extras`, `dotfile verify` run **unchanged**. The
  `~/.config` dotfiles, zsh/tmux plugins, and verify checks still apply — the
  tools they check for are on PATH via `systemPackages`.
- `set_zsh_default`: skip `chsh` on NixOS (the shell is set declaratively in the
  config). One platform guard.

### 5. Tests — `tests/bash/`

- `test_platform.sh`: `detect_platform` returns `nixos` when `/etc/os-release`
  has `ID=nixos` (mock the file).
- `test_packages.sh`: `install_nixos` in `DRY` mode logs the rebuild and does not
  call the github-release installers.
- `test_cli.sh`: `--dry packages` on a mocked NixOS exits 0 and does no work.

## Out of scope (later)

- Flakes (`flake.nix` + `flake.lock`, `nixos-rebuild switch --flake`).
- Porting app configs (hyprland/waybar/etc.) into Nix modules.
- home-manager.
- Boot-testing the config on real hardware — the template will need tuning.
